Історія змін

Як було сказано на початку, основною функцією системи контролю версій є зберігання та ведення історії внесених змін. На попередніх уроках ми вже внесли декілька, тому варто навчитись їх переглядати. Для перегляду історії змін (логу) у Git існує відповідна команда:

bash

git log

$

Розглянемо вивід цієї команди. Як бачимо, вона виводить список збережених комітів з відповідними повними контрольними сумами, які унікально їх ідентифікують, повідомленнями, що супроводжують кожен коміт, а також такими деталями, як дані про автора та дату і час збереження коміту.

В залежності від бажаної деталізації виводу команди пропонуються різні параметри. Якщо користувач бажає побачити, які саме зміни були внесені в файли з кожним комітом, варто використовувати параметр -p або --patch. Також можна обмежити кількість комітів, зміни яких будуть виведені. Наприклад, щоб переглянути деталізацію змін в останніх двох комітах, виконуємо таку команду:

bash

git log -p -2

(FIXME вывод)

$

Вивід команди git log можна гнучко налаштовувати за допомогою параметра --pretty. Наприклад, якщо необхідно переглянути стислу історію «в один рядок», можна скористатись командою:

bash

git log --pretty=oneline

(FIXME OUTPUT)

$
