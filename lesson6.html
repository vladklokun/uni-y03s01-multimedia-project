Налаштування репозиторію

Перш за все, варто організувати створений репозиторій для комфортної роботи.

Опис проекту

Розпочнемо з текстового файлу, який описує проект: створюємо файл README.md за допомогою такої команди:

bash

echo This is an example Python project for showcasing Git basics. >> README.md

$

Ліцензія

Створивши стислий опис проекту, варто обрати для нього ліцензію, яка регулюватиме права та обов'язки користувачів. Для цього тестового проекту була обрана ліцензія Creative Commons 0 1.0 Universal, тому створимо відповідний файл LICENSE.md та заповнимо його текстом обраної ліцензії будь-яким зручним способом, наприклад, вручну скопіювавши текст обраної ліцензії.

Файлова структура проекту

Тепер створимо скелет проекту на Python. Це не входить в кроки налаштування Git-репозиторію, але створення правильної файлової структури проекту є гарною практикою. Для нашого тестового проекту на Python необхідна досить проста структура, створимо її:

bash

mkdir docs test pyhello

$ touch docs/docfile.txt test/unneeded-test.py pyhello/pyhello.py

$

Директорія docs міститиме документацію до проекту, test — тести для розробленої програми, а pyhello — безпосередньо початкові коди тестового проекту.

Ігнорування небажаних файлів

У процесі розробки, зборки та тестування проектів можуть з'являтись файли, які необхідні лише локальному розробнику, але їх відстеження в репозиторії небажане, тому такі файли прийнято ігнорувати. Зазвичай це файли певних розширень, або з характерними назвами, тому їх зручно ігнорувати за допомогою регулярних виразів. В Git це реалізується за допомогою спеціального файлу .gitignore, що містить регулярні вирази для пошуку непотрібних файлів та директорій за їх назвами. Створимо файл .gitignore та заповнимо його необхідними даними з коментарями:

bash

# Скомпільовані об'єкти

__pychache__/ # тепер директорія __pychache__ повністю ігнорується

*.py[cod] # ігнорувати файли з будь-яким ім'ям та розширеннями «pyc», «pyo» і «pyd»

# Розширення мови C

*.so # ігнорувати усі файли з розширенням «so»

# Лог-файли інсталяторів

pip-log.txt # ігнорувати тільки файл «pip-log.txt»

pip-delete-this-directory.txt # ігнорувати тільки файл «pip-delete-this-directory.txt»

# Документація Sphinx

docs/_build/ # в директорії «build» ігнорувати субдиректорію «_build»

# ...і так далі

Якщо ви не хочете, щоб Git відстежував будь-які файли, просто додайте їх назву або регулярний вираз, який знаходить небажаний клас файлів, до файлу .gitignore.

Налаштування атрибутів

Система Git розрахована на роботу з файлами будь-яких типів, однак, найзручніше працювати саме з текстовими початковими файлами. Незважаючи на це, у процесі створення проекту може виникнути потреба зберігати двійкові та інші нетекстові файли у репозиторії з різних причин. Для запобігання небажаних змін у двійкових файлах зручно явно визначити тип файлу — двійковий або текстовий — за розширенням. Це реалізується за допомогою файлу .gitattributes, який містить вказівки для присвоєння атрибутів певним класам файлів.

Формат інструкції такий:

bash

<регулярний вираз для пошуку файлу> <атрибути>

Створимо та заповнимо файл .gitattributes за наведеним форматом з коментарями:

bash

*.gitattributes text # позначити файли з розширенням «gitattributes» як текстові

.gitignore text # файл «.gitignore» — текстовий

*.md text # усі файли з розширенням «md» — текстові

*.txt text # усі файли з розширенням «txt» — текстові

Існує й багато інших кроків для підготовки Git-репозиторію, але вони виходять за рамки масштабу даного курсу.
