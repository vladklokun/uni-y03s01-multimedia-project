Що таке Git?

Git — це розподілена система контролю версій, яку розробив Лінус Торвальдс, коли працював над ядром Linux, бо був незадоволений існуючими системами контролю версій. На момент розробки цього курсу за даними многочислених опитувань (наприклад, StackOverflow Developer Survey) саме Git є найпопулярнішою системою контролю версій.

Як Git зберігає дані?

На відміну від інших систем контролю версій, Git зберігає не зміни, внесені до контрольованих файлів, а повні «знімки» версій цих файлів, ніби файлова система. Такий підхід впливає на багато інших аспектів роботи програми, тому бути зустрічатись у наступних уроках.

Майже усі операції локальні

Більшості операцій у Git для роботи потрібні лише локальні файли та ресурси, тому зникає залежність від третіх сторін. В результаті зростає загальна швидкість виконання операцій, оскільки вони виконуються лише на локальній машині і залежать виключно від неї. Також з'являється можливість роботи, внесення та збереження змін у локальному репозиторії без обмежень, незалежно від того, чи підключений користувач до мережі.

Git перевіряє цілісність

Цілісність даних є невід'ємною частиною Git, тому перевірка даних на цілісність вбудована на найнижчих рівнях. Перед зберіганням у репозиторії, Git обчислює контрольну суму об'єкта та використовує її як ідентифікатор. Завдяки цьому неможливо змінити вміст будь-якого відстежуваного файлу або директорії так, щоб Git цього не помітив, що дозволяє виявляти пошкодження файлів при зберіганні та передачі.

Git зазвичай лише додає дані

Майже усі дії з Git лише додають дані до бази (репозиторію), тому дуже важко стерти відстежувані дані або зробити щось, що не можна відмінити. Це стосується лише відстежуваних даних, які були додані до репозиторія операцією `commit`, і модифіковані дані, які не відстежуються, легко можуть бути втрачені, як і у інших системах контролю версій. Однак, після того як дані були додані у репозиторій, їх дуже важко втратити, особливо якщо користувач регулярно зберігає базу даних в іншому репозиторії.

Три стани

Основним поняттям для ясного розуміння процесу роботи Git є три основні стани, в яких можуть знаходитись файли: 1. Збережений у коміті (commited) — означає, що файли безпечено збережені у локальній базі даних. 2. Змінений (modified) — означає, що файл змінений порівняно з версією у локальній базі даних. 3. Індексований (staged) — означає, що зміни, внесені у файл, позначені для додавання у наступний коміт. Рис. з трьома станами.

Директорія Git — це найважливіша частина репозиторію, в ній зберігаються метадані та база даних об'єктів проекту. Саме ця директорія копіюється під час клонування репозиторію з інших комп'ютерів.

Таким чином, базовий процес використання Git можна описати 3 кроками:

1. Користувач модифікує файли у робочій області.

2. Користувач вибірково індексує лише ті зміни, які хоче додати у наступному коміті, що додає виключно обрані зміни до індексованої області.

3. Користувач підтверджує зміни — виконує коміт, в результаті чого Git робить знімок поточної версії файлів в індексованій області та зберігає зроблений знімок у директорії Git.

Версія файлу, що знаходяться у директорії Git, вважається збереженою у коміті. Якщо до такої версії внесли зміни, вона вважається модифікованою. Якщо модифіковану версію додали до індексованої області, вона вважається індексованою.
