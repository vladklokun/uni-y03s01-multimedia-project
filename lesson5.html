Створення репозиторію проекту за допомогою Git

Репозиторій — це загальна назва місця збереження файлів проекту та важливих метаданих в Git. Зазвичай користувач отримує репозиторій 2 способами:

1. Перетворює локальну директорію, яка наразі не відстежується системою контролю версій, у Git-репозиторій.

2. Клонує існуючий Git-репозиторій з будь-якого іншого місця.

В обох випадках користувач одержує готову до роботи локальну копію Git-репозиторію.

У рамках цього курсу ми розглядатимемо покрокову роботу з локальним репозиторієм Git на прикладі тестового проекту pyHello на мові Python у середовищі командного рядка bash.

Створення репозиторію

Перше, що необхідно зробити перед початком роботи над проектом, це створити директорію для нього, тому виконуємо команду:

bash

mkdir pyhello

$

Далі необхідно ініціалізувати власне репозиторій Git, тому переходимо у створену директорію та виконуємо ініціалізацію:

bash

cd pyhello

$ git init

$

Тепер Git ініціалізований для директорії pyhello, але зараз нічого не відстежує.
